Create Some Default Data

First, let's create some text data to collaborate on. When creating a new RealTimeModel you need to initialize the model with initial data. In a previous step, we created a placeholder file tutorial-data.js where will now create the default data. Paste the following code in the tutorial-data.js file.

Open the Model

Next we will update the tutorial.js to open the model after the client connects, instead of simply logging to the console. We will use the openAutoCreate method of the ModelService to open a RealTimeModel. The openAutoCreate method will open a model if it exists, or create it if it does not exist. We will use the "convergence-tutorials" collection and the id of "textarea". All clients will use the some id, thus they will all open the same model. The ephemeral option is set to true, so that when the last user closes the model, it will be deleted. The data option supplies the default data that will be used to initialize the model when the first user opens it.

tutorial.js

Convergence.connectAnonymously(CONVERGENCE_URL, username).then(domain => {
  return domain.models().openAutoCreate({
    collection: "convergence-tutorials",
    id: "textarea",
    ephemeral: true,
    data: {
      text: DEFAULT_TEXT_DATA
    }
  });
}).then(model => {
  console.log("The model is open!")
}).catch(error => {
  console.error(error);
});

Checkpoint

At this point you should be able to refresh your browser tab that contains the index.html and see the text "The model is open!" printed to the console. If there is an error, check your code and adjust as necessary.
