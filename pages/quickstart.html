Get started with your first collaborative application.

Get Convergence

We're in the process of getting the Convergence server images published to Docker Hub. In the meantime, fill out this form and we'll let you know when they're ready!

Get the Code

To get the code, clone or fork the Getting Started Repository from GitHub: 

git clone https://github.com/convergencelabs/model-getting-started.git

Prefer a more hands-on approach? Follow our more in-depth tutorial here.

Then, configure the application to connect to your domain using the user you just created. Copy config.example.js to config.js and edit the DOMAIN_URL value to what you saw in the Administration UI in the previous step.

Collaborate

Simply open index.html in your web browser to run the application. Open it a second time in another web browser and type in the textarea to see your changes propagated in real time!

Bonus: Back in the Administration console, click "Models" in the sidebar of the domain area, and click into the "getting-started" model. This is a real-time view of the contents of the model. The data there will update immediately as you type in the textarea for easy, visual debugging.

Want more? Check out the examples repository in GitHub demonstrating activities, chat, and more.

  1. Domains are organizational entities: you can think of them as projects, different environments (such as Prod or Staging), or some combination of both. Domains are self-encapsulated, configurable, and contain their own data and users. See here for more information.
